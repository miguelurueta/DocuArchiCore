## Requirements
### Requirement: Traceability to Jira issue
The system MUST keep traceability between this OpenSpec change and Jira issue SCRUM-101.

#### Scenario: Change references Jira issue
- **WHEN** a reviewer opens the change artifacts
- **THEN** proposal and design identify Jira issue SCRUM-101

### Requirement: Backend update rules baseline
Backend update requests MUST follow repository, architecture and testing constraints defined in openspec/context/OPSXJ_BACKEND_RULES.md.

#### Scenario: Missing implementation constraints
- **WHEN** proposal/design/tasks are reviewed
- **THEN** they explicitly include route confirmation, interface policy, DI registration, AppResponses/try-catch and test requirements

### Requirement: RegistrarRadicacionEntranteAsync exposes Q08 last insert
The system MUST propagate the identifier generated by `ExecuteQ08InsertEstadoInicialAsync` through repository and service contracts.

#### Scenario: Successful Q08 insert returns idEstadoRadicado
- **GIVEN** `RegistrarRadicacionEntranteAsync` executes Q08 successfully
- **WHEN** the repository builds `RegistrarRadicacionEntranteResponseDto`
- **THEN** `ReturnRegistraRadicacion.IdEstadoRadicado` contains the `LAST_INSERT_ID()` from `ra_rad_estados_modulo_radicacion`

#### Scenario: Legacy response is normalized from metadata
- **GIVEN** a successful repository response without `IdEstadoRadicado` in `ReturnRegistraRadicacion`
- **WHEN** metadata includes `idEstadoRadicado`
- **THEN** the service normalizes the response and keeps the flow compatible
