## Requirements
### Requirement: Traceability to Jira issue
The system MUST keep traceability between this OpenSpec change and Jira issue SCRUM-119.

#### Scenario: Change references Jira issue
- **WHEN** a reviewer opens the change artifacts
- **THEN** proposal and design identify Jira issue SCRUM-119

### Requirement: Backend update rules baseline
Backend update requests MUST follow repository, architecture and testing constraints defined in `openspec/context/OPSXJ_BACKEND_RULES.md`.

#### Scenario: Missing implementation constraints
- **WHEN** proposal, design and tasks are reviewed
- **THEN** they explicitly include route confirmation, interface policy, DI registration, AppResponses/try-catch and test requirements

### Requirement: Structured filters affect workflow inbox queries
The system MUST translate `StructuredFilters` from `WorkflowInboxApiRequestDto` into additional safe conditions for the workflow inbox query built by `WorkflowInboxQueryBuilder`.

#### Scenario: Structured filters apply over allowed dynamic columns
- **GIVEN** a valid inbox request with `StructuredFilters`
- **AND** each `Field` matches either a dynamic metadata column or one of `fecha_inicio`, `ESTADO`, `id_tarea`
- **WHEN** `WorkflowInboxQueryBuilder.Build` generates `QueryOptions`
- **THEN** the final query includes the structured filter conditions
- **AND** the existing workflow base filters remain intact

#### Scenario: Structured filters combine with search type 1
- **GIVEN** an inbox request with `SearchType = 1`
- **AND** one or more valid `StructuredFilters`
- **WHEN** query options are built
- **THEN** the query applies the structured filter conditions in addition to the base workflow filters

#### Scenario: Structured filters combine with search type 2
- **GIVEN** an inbox request with `SearchType = 2`
- **AND** one or more valid `StructuredFilters`
- **WHEN** query options are built
- **THEN** the query preserves the global LIKE search behavior
- **AND** it also applies the structured filter conditions

#### Scenario: Structured filters combine with search type 3
- **GIVEN** an inbox request with `SearchType = 3`
- **AND** one or more valid `StructuredFilters`
- **WHEN** query options are built
- **THEN** the query preserves the controlled advanced search expression
- **AND** it also applies the structured filter conditions

### Requirement: Structured filters enforce operator and field safety
The system MUST reject structured filters that use unsupported fields or unsupported operators.

#### Scenario: Field is outside the whitelist
- **GIVEN** a `StructuredFilter` whose `Field` does not map to an allowed static field or resolved dynamic metadata column
- **WHEN** the query builder processes the request
- **THEN** the operation fails in a controlled way
- **AND** no unsafe SQL condition is generated

#### Scenario: Operator is unsupported
- **GIVEN** a `StructuredFilter` whose `Operator` is not one of `eq`, `neq`, `contains`, `startsWith`, `endsWith`, `gt`, `gte`, `lt`, `lte`, `between`, `isNull`, `isNotNull`
- **WHEN** the query builder processes the request
- **THEN** the operation fails in a controlled way
- **AND** no unsafe SQL condition is generated

### Requirement: Structured filters respect field data types
The system MUST validate operator compatibility against the resolved field data type before generating the final condition.

#### Scenario: Text-only operator over non-text field
- **GIVEN** a `StructuredFilter` with operator `contains`, `startsWith` or `endsWith`
- **AND** the resolved field is not text
- **WHEN** the query builder processes the request
- **THEN** the operation fails in a controlled way

#### Scenario: Comparable operator over unsupported field type
- **GIVEN** a `StructuredFilter` with operator `gt`, `gte`, `lt`, `lte` or `between`
- **AND** the resolved field is neither numeric nor date/datetime
- **WHEN** the query builder processes the request
- **THEN** the operation fails in a controlled way

### Requirement: Structured filters sanitize values before SQL generation
The system MUST sanitize structured filter values before appending them to the controlled SQL conditions generated by `WorkflowInboxQueryBuilder`.

#### Scenario: Text value contains apostrophes
- **GIVEN** a valid text structured filter value containing apostrophes
- **WHEN** the query builder generates the condition
- **THEN** the text literal is escaped safely

#### Scenario: Numeric or date value is invalid
- **GIVEN** a structured filter with a numeric or date operator
- **AND** the provided value cannot be parsed for the resolved field type
- **WHEN** the query builder processes the request
- **THEN** the operation fails in a controlled way
