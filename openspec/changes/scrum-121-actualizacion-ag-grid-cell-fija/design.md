## Problem Statement

PROMPT ARQUITECTÓNICO Soportar columnas fijas (pinned) en la línea dinámica de AppTable Rol esperado: Arquitecto de software senior y desarrollador frontend React (React 19 + TypeScript estricto + AG Grid + arquitectura enterprise) OBJETIVO Extender la línea dinámica de AppTable para que columnas provenientes del contrato backend puedan declararse como fijas a la izquierda o a la derecha, preservando esa metadata hasta el ColDef final consumido por AG Grid. La solución debe permitir que: backend declare columnas fijas con Pinned frontend preserve esa metadata en la normalización intermedia AppTable reciba ColDef final con pinned se mantenga compatibilidad con la API manual actual de AppTable CONTEXTO Hoy AppTable ya soporta columnas fijas si recibe ColDef<T>[] manualmente con: pinned: "left" | "right" lockPinned: true Pero la línea dinámica actual: DynamicUiTableDto -> AppGridColumn -> ColDef no transporta esa metadata, por lo que las tablas dinámicas no pueden fijar columnas desde backend. ARQUITECTURA A RESPETAR Backend DynamicUiTableDto -> dynamicUiToAgGridColumns -> AppGridColumn -> appGridToAppTableColumns -> ColDef -> AppTable Reglas: no modificar la API manual actual de AppTable no acoplar la solución a una pantalla específica no introducir lógica de dominio no romper contratos dinámicos existentes ALCANCE Debe: extender el contrato dinámico de columnas para soportar Pinned extender el modelo interno AppGridColumn propagar Pinned y LockPinned desde el DTO backend mapear esa metadata a ColDef.pinned y ColDef.lockPinned agregar pruebas de mapeo y regresión NO ALCANCE no rediseñar AppTable no cambiar la API manual de columns: ColDef<T>[] no introducir lógica específica de gestionCorrespondencia no tocar actions salvo que la columna de acción requiera convención de pinned opcional UBICACIÓN ESPERADA Archivos a ajustar: src/app/Components/UI/AppTable/types/dynamicUiTable.types.ts src/app/Components/UI/AppTable/adapters/dynamicUiToAgGridColumns.ts src/app/Components/UI/AppTable/adapters/appGridToAppTableColumns.ts Tests esperados: src/app/Components/UI/AppTable/tests/dynamicUiToAgGridColumns.test.ts src/app/Components/UI/AppTable/tests/appGridToAppTableColumns.test.ts CONTRATO DINÁMICO ESPERADO En UiColumnDto Se debe soportar: Pinned?: "left" | "right" | null pinned?: "left" | "right" | null LockPinned?: boolean | null lockPinned?: boolean | null En AppGridColumn Se debe agregar: pinned?: "left" | "right" lockPinned?: boolean REGLAS DE MAPEADO 1. Backend -> AppGridColumn dynamicUiToAgGridColumns.ts debe: leer column.pinned ?? column.Pinned leer column.lockPinned ?? column.LockPinned preservar esos valores en AppGridColumn 2. AppGridColumn -> ColDef appGridToAppTableColumns.ts debe: mapear column.pinned a colDef.pinned mapear column.lockPinned a colDef.lockPinned 3. Fallback si la metadata no existe, no aplicar pinning no inventar defaults para columnas normales REGLAS DE COMPATIBILIDAD columnas manuales con ColDef.pinned siguen funcionando igual columnas dinámicas sin Pinned no cambian comportamiento no romper SCRUMCORE-35, SCRUMCORE-36 ni SCRUMCORE-37 OPCIONAL RECOMENDADO Si la arquitectura del producto lo permite, se puede definir una convención reusable para isActionColumn: pinned = "right" lockPinned = true suppressMovable = true Pero esto debe ser: explícito reusable no acoplado a una sola pantalla Si se implementa, debe quedar documentado y cubierto por tests. REGLAS DE INMUTABILIDAD no mutar UiColumnDto no mutar AppGridColumn construir nuevos objetos en cada transformación PRUEBAS OBLIGATORIAS Cubrir mínimo: mapeo de Pinned = "left" desde DTO mapeo de Pinned = "right" desde DTO preservación de LockPinned mapping correcto a ColDef columna sin pinning no cambia si se implementa convención de isActionColumn, cubrirla explícitamente CRITERIOS DE ACEPTACIÓN una columna dinámica puede quedar fija a la izquierda o derecha desde metadata backend AppTable sigue consumiendo ColDef final sin cambios de API la línea dinámica preserva correctamente Pinned y LockPinned no se rompen tablas actuales la solución queda reusable para cualquier tabla dinámica INSTRUCCIÓN FINAL Antes de implementar: validar contrato actual de UiColumnDto validar adapter dynamicUiToAgGridColumns.ts validar adapter appGridToAppTableColumns.ts confirmar que AppTable no necesita cambios de API Luego: implementar con TypeScript estricto mantener desacoplamiento no introducir lógica de dominio Finalmente reportar: cambios de tipos estrategia de propagación mapping final a ColDef compatibilidad preservada decisión sobre convención opcional para isActionColumn
