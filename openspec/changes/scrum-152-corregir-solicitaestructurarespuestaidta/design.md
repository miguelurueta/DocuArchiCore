## Problem Statement

PROMPT ARQUITECTÓNICO — Ticket BE (LISTO PARA COPIAR/PEGAR) Título Corregir SolicitaEstructuraRespuestaIdTareaAsync para mapear correctamente ID_TAREA_WF y campos clave (evitar valores vacíos por mapeo) Rol esperado Arquitecto backend senior (.NET, C#, Core, Dapper/engine interno, mapeo DB→modelo, pruebas, documentación técnica). ——— OBJETIVO Corregir el caso de uso SolicitaEstructuraRespuestaIdTareaAsync para que el resultado devuelto por: IServiceSolicitaEstructuraRespuesta.SolicitaEstructuraRespuestaIdTareaAsync(...) incluya correctamente el valor de ID_TAREA_WF (mapeado a IdTareaWf) y los campos críticos usados por tokens/plantillas, evitando que lleguen null/0 cuando en BD sí existen. Este ticket es prerequisito para el módulo de hidratación de plantilla/tokens (GetInitialContentAsync), ya que los tokens dependen de datos consistentes. ——— PROBLEMA Actualmente el repositorio filtra por ra_respuesta_radicado.ID_TAREA_WF, pero el objeto materializado (RaRespuestaRadicado o DTO equivalente) llega con IdTareaWf vacío/null. La tabla ra_respuesta_radicado contiene ID_TAREA_WF (BIGINT), por lo que el problema es de mapeo DB→modelo (DB legacy UPPER_SNAKE_CASE vs propiedades C#). Causa probable: Se usa IDapperCrudEngine.GetAllAsync<T> (o mapeo automático) con modelo/atributos [Column] que no coinciden con los nombres físicos (ID_TAREA_WF, ID_AREA, etc.), y el motor no aplica conversión ni respeta [Column]. ——— CAMBIO REQUERIDO (CRÍTICO) Ajustar el repositorio para usar SQL explícito con alias (independiente de si se ejecuta vía Dapper directo o engine interno), de modo que el mapeo no dependa de convenciones implícitas: rr.ID_TAREA_WF AS IdTareaWf (OBLIGATORIO) Alias explícitos para campos usados por tokens/plantilla/flujo (mínimo): rr.RADICADO AS Radicado rr.DESTINATARIO AS Destinatario rr.DIRECCION_DESTINATARIO AS DireccionDestinatario rr.ASUNTO AS Asunto rr.USUARIO_RESPONSABLE AS UsuarioResponsable rr.CARGO_RESPONSABLE AS CargoResponsable rr.AREA_RESPONSABLE AS AreaResponsable (y cualquier otro campo que el flujo actual consuma) Reglas: SQL parametrizado (sin concatenación). Mantener filtro: WHERE rr.ID_TAREA_WF = @idTareaWf. Mantener contrato y semántica actual de AppResponses. Nota: No es obligatorio mapear todas las columnas de la tabla; sí es obligatorio mapear IdTareaWf y los campos que alimentan tokens/ plantillas. ——— CONTRATO (SE MANTIENE) Task<AppResponses<List<RaRespuestaRadicado>>> SolicitaEstructuraRespuestaIdTareaAsync(long idTareaWf, string defaultDbAlias) Semántica: Con filas: success=true, message="YES", data=[...] Sin filas: success=true, message="Sin resultados", data=[] Validación: success=false con errores de validación Excepción: success=false con error tipo excepción (estándar del proyecto) ——— UBICACIÓN (ARCHIVOS) Repositorio: MiApp.Repository/Repositorio/GestionCorrespondencia/SolicitaEstructuraRespuestaIdTareaRepository.cs Servicio: MiApp.Services/Service/GestorDocumental/ServiceSolicitaEstructuraRespuesta.cs Controlador: DocuArchi.Api/Controllers/GestionCorrespondencia/SolicitaEstructuraRespuestaIdTareaController.cs Modelo/DTO: MiApp.Models/Models/GestionCorrespondencia/RaRespuestaRadicado.cs ——— PRUEBAS OBLIGATORIAS Unitarias (mínimo) Validaciones: defaultDbAlias inválido → error controlado idTareaWf <= 0 → error controlado Integración (preferida; obligatoria si existe infraestructura) Dada una fila con ID_TAREA_WF = X: al consultar X, data.Count > 0 data.All(r => r.IdTareaWf == X) (o, como mínimo, IdTareaWf no vacío y consistente) campos mínimos para tokens (Radicado, Destinatario, Asunto, etc.) llegan poblados si existen en BD Sin filas: success=true, data=[], message="Sin resultados" Si no hay BD de pruebas, documentar explícitamente la limitación y dejar preparado el test para ejecución en ambiente con DB. ——— DOCUMENTACIÓN (ACTUALIZAR) Crear/actualizar: SCRUM-[ID]-Implementacion-Detallada-SolicitaEstructuraRespuestaIdTarea.md Incluir: problema (mapeo) decisión (SQL explícito con alias) snippet de SQL (sin credenciales) evidencia de pruebas ——— CRITERIOS DE ACEPTACIÓN SolicitaEstructuraRespuestaIdTareaAsync retorna IdTareaWf correcto cuando existe en BD.
