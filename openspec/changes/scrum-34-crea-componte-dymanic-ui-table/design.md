## Problem Statement

PROMPT PROFESIONAL — Backend “DynamicUiTableBuilder” (Reusable) + Endpoints Genéricos y Específicos (DTO-in/DTO-out) Proyecto: MiApp ( Core Web API, multirepo) Rol: Arquitecto de Software + Backend Senior Objetivo: Implementar un componente reusable backend que construya la “estructura de tabla UI dinámica” (DTO + acciones) para React/MUI, soportando endpoints genéricos y específicos por módulo, sin amarrar el diseño a una tabla concreta. ===================================================================== CONTEXTO / PROBLEMA Necesitamos un backend que permita al frontend (React/MUI-GRID-TABLE) renderizar tablas configurables sin reescribir lógica por entidad. La UI debe recibir configuración de columnas, datos paginados, acciones (globales, masivas, por fila y por celda), y el backend debe controlar autorización por claims y seguridad (SQLi). ===================================================================== OBJETIVOS Unificar construcción de DTO de tabla dinámica para cualquier TableId. Soportar: Columnas desde DB (configurable por tabla/módulo) Columnas fijas definidas por código (modo estructura fija) Columas definidas en al backend y configuración de backEnd. Acciones: Toolbar actions, Bulk actions, Row actions, Cell actions Seguridad: Claims: RequiredClaimsAny (visible) / RequiredClaimsAll (ejecutable) Backend revalida claims siempre al ejecutar acción Sorting con whitelist para evitar SQL injection Queries parametrizadas y validaciones de entrada ===================================================================== STACK / RESTRICCIONES (OBLIGATORIAS) Core Web API Clean Architecture: Controller → Service → Repository DapperCrudEngine + QueryOptions (usar la clase real del proyecto) Wrapper de respuesta: AppResponses<T> (success/message/data/meta/errors) NO pseudocódigo / NO TODOs / NO “magic strings” (usar constantes cuando aplique) Código modular, reutilizable, consistente y testeable Manejo estándar de errores con try/catch en todas las funciones públicas ===================================================================== UBICACIONES MÍNIMAS (OBLIGATORIO) DTOs: MiApp.DTOs/DTOs/UI/MuiTable Services: MiApp.Service/Service/UI/MuiTable Repos: MiApp.Repository/Repositorio/UI/MuiTable Docs/Diagramas: /Docs/UI/MuiTable ===================================================================== CONTRATO DE RESPUESTA (OBLIGATORIO) Todas las respuestas deben retornar AppResponses<T>: success: bool message: string data: T meta: AppMeta? errors: AppError[]?
