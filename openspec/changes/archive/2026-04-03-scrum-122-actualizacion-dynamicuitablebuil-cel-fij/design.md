## Problem Statement

PROMPT ARQUITECTÓNICO Agregar soporte de Pinned y LockPinned en UiColumnDto para columnas dinámicas de AppTable Rol esperado: Arquitecto de software senior y desarrollador backend .NET OBJETIVO Extender el contrato backend de columnas dinámicas para que DynamicUiTableDto.Columns pueda declarar columnas fijas a la izquierda o derecha, alineado con el soporte ya implementado en frontend en la línea dinámica de AppTable. El backend debe permitir que cada UiColumnDto pueda definir: Pinned LockPinned de modo que frontend pueda preservar esa metadata hasta ColDef.pinned y ColDef.lockPinned. CONTEXTO Frontend ya soporta esta metadata en la línea dinámica: DynamicUiTableDto -> AppGridColumn -> ColDef -> AppTable Pero el backend todavía debe poblarla en UiColumnDto para que las tablas dinámicas puedan fijar columnas desde la respuesta API. ARQUITECTURA A RESPETAR Backend -> DynamicUiTableDto -> Frontend dinámico -> AppTable Reglas: mantener UiColumnDto como contrato base de columnas dinámicas no romper consumidores actuales no acoplar el cambio a una sola tabla o módulo dejar la solución reusable para cualquier tabla dinámica ALCANCE Debe: extender UiColumnDto para soportar Pinned extender UiColumnDto para soportar LockPinned permitir que cualquier generador de DynamicUiTableDto pueda poblar esos campos mantener compatibilidad con respuestas que no usen pinning Aplica Pinned a workflowInboxgestion NO ALCANCE no rediseñar DynamicUiTableDto no tocar acciones dinámicas no imponer pinning a todas las tablas no introducir lógica de frontend en backend no acoplar el cambio a workflowInboxgestion solamente UBICACIÓN ESPERADA Contrato principal: MiApp.DTOs.DTOs.UI.MuiTable.UiColumnDto Y, si aplica, ajustes en: builders/factories/mappers que construyen DynamicUiTableDto.Columns tests del backend para serialización o composición de columnas CONTRATO ESPERADO En UiColumnDto agregar: public string? Pinned { get; set; } public bool? LockPinned { get; set; } REGLAS DE CONTRATO Pinned Valores válidos: "left" "right" null o vacío cuando no aplique No permitir otros valores arbitrarios. LockPinned Valores válidos: true false null REGLAS FUNCIONALES si una columna debe fijarse a la izquierda, backend envía Pinned = "left" si una columna debe fijarse a la derecha, backend envía Pinned = "right" si además no debe poder desanclarse, backend envía LockPinned = true si no aplica pinning, ambos campos pueden omitirse COMPATIBILIDAD respuestas existentes que no incluyan Pinned o LockPinned deben seguir funcionando igual no se debe obligar a poblar esos campos en todos los productores de UiColumnDto el cambio debe ser backward-compatible EJEMPLO ESPERADO { "TableId": "workflowInboxgestion", "Columns": [ { "DataIndex": "RADICADO", "HeaderName": "Radicado", "Pinned": "left", "LockPinned": true }, { "DataIndex": "ASUNTO", "HeaderName": "Asunto" }, { "DataIndex": "acciones", "HeaderName": "Acciones", "Pinned": "right", "LockPinned": true } ] } VALIDACIONES no emitir valores distintos de left o right no romper serialización actual de UiColumnDto mantener compatibilidad con normalización actual del frontend si existen validadores de DTO, contemplar los nuevos campos PRUEBAS OBLIGATORIAS Cubrir mínimo: serialización de Pinned = left serialización de Pinned = right serialización de LockPinned = true columna sin pinning sigue serializando igual un DynamicUiTableDto puede incluir mezcla de columnas con y sin pinning CRITERIOS DE ACEPTACIÓN UiColumnDto soporta Pinned UiColumnDto soporta LockPinned el backend entrega esos valores en DynamicUiTableDto.Columns columnas sin esos campos siguen funcionando igual el contrato queda reusable para cualquier tabla dinámica INSTRUCCIÓN FINAL Antes de implementar: validar contrato actual de UiColumnDto validar productores de DynamicUiTableDto validar si existe capa de validación/normalización DTO Luego: implementar sin romper compatibilidad mantener contrato simple y reusable no introducir semántica de frontend más allá de Pinned/LockPinned Finalmente reportar: cambios en UiColumnDto impacto en serialización productores ajustados compatibilidad preservada ejemplo de payload final
