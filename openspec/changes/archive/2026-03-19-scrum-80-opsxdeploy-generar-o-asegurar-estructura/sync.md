Use this template at the start of each Jira ticket (`SCRUM-80`) to decide exactly where to run `opsxj:new` and `opsxj:archive`.

## Operating Rule

Run `opsxj:new SCRUM-80` only in rows with `Impacta? = yes`.
Run `opsxj:archive SCRUM-80` only after the repo PR is merged.
Rows with `Impacta? = no` must stay as `n/a` to make scope explicit.

## Typical API Change Pattern

- `DocuArchi.Api`: usually yes
- `MiApp.Services`: usually yes
- `MiApp.Repository`: usually yes
- `MiApp.DTOs`: usually yes
- `MiApp.Models`: optional (only if model changes)
- `DocuArchiCore` / `DocuArchiCore.Web`: only if coordinator or UI is in scope
