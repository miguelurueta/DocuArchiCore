## Problem Statement

Descripción: Se requiere extender opsxdeploy para que el flujo de empaquetado para IIS pueda dejar un web.config con estructura base válida para Core cuando el archivo no exista, y validar la estructura mínima cuando sí exista. El objetivo es reducir errores manuales al preparar publicaciones para IIS y asegurar que el paquete generado tenga una base operativa consistente para despliegue. La solución no debe hacer obligatorio el despliegue directo a IIS, pero sí debe dejar el paquete listo con una configuración base estandarizada. Además, la actualización de la documentación debe ser un requisito obligatorio del ticket, tanto en la documentación del tool como en el manual operativo de publicación. Objetivo: Permitir que opsxdeploy garantice una estructura mínima útil de web.config para paquetes IIS, manteniendo un comportamiento seguro frente a archivos existentes y dejando documentación actualizada como parte obligatoria del entregable. Alcance: extender opsxdeploy:publish-package generar web.config base cuando no exista validar estructura de web.config existente no hacer obligatorio desplegar directamente a IIS actualizar documentación del tool y manual operativo Reglas funcionales: Si el publish no contiene web.config, el sistema debe generar un archivo base compatible con AspNetCoreModuleV2. Si el publish ya contiene web.config, el sistema no debe sobreescribirlo automáticamente en esta fase inicial. Si existe web.config, el sistema debe validar como mínimo: existencia de aspNetCore processPath arguments bloque environmentVariables o advertencia de ausencia El sistema debe seguir permitiendo generar un paquete limpio sin desplegar directamente al sitio IIS. El nombre del ensamblado usado en arguments debe poder derivarse del publish si es posible. La estructura base debe incluir placeholders para variables críticas de entorno. Estructura mínima esperada del web.config generado: configuration location system.webServer handlers aspNetCore environmentVariables Variables placeholder esperadas: ASPNETCORE_ENVIRONMENT ConnectionStrings__MySqlConnection_DA ConnectionStrings__MySqlConnection_WFR ConnectionStrings__MySqlConnection_WF Jwt__Key Jwt__Issuer Jwt__Audience StoragePaths__Temp StoragePaths__Uploads StoragePaths__Avatars StoragePaths__Exports StoragePaths__Logs Criterios de aceptación: opsxdeploy:publish-package debe generar un web.config base cuando el archivo no exista en la carpeta de publicación. El web.config generado debe ser válido para una aplicación Core hospedada en IIS con AspNetCoreModuleV2. Si ya existe web.config, opsxdeploy no debe sobrescribirlo automáticamente. Si ya existe web.config, opsxdeploy debe reportar si falta estructura mínima requerida. El paquete generado debe seguir excluyendo artefactos de desarrollo y tooling no productivo. La documentación de Tools/iis-deploy/README.md debe actualizarse para explicar el nuevo comportamiento. El manual operativo Docs/Publicacion/IIS-DocuArchiApi.md debe actualizarse obligatoriamente para incluir el manejo de web.config. Las pruebas automatizadas deben cubrir: generación de web.config cuando falta conservación de web.config existente validación/reportes sobre estructura mínima El flujo debe seguir permitiendo empaquetado para IIS sin requerir despliegue directo. Entregables obligatorios: cambio en Tools/iis-deploy/opsxdeploy.ps1 actualización de pruebas del tool actualización de Tools/iis-deploy/README.md actualización del manual Docs/Publicacion/IIS-DocuArchiApi.md Subtareas sugeridas: Definir estrategia de generación segura de web.config Implementar derivación de arguments desde el ensamblado principal Implementar generación de plantilla base Implementar validación de web.config existente Ajustar publish-package para incluir esta lógica Agregar pruebas automatizadas Actualizar README del tool Actualizar manual operativo de publicación IIS Notas: En esta fase no se debe reescribir automáticamente un web.config ya existente. La documentación actualizada es requisito obligatorio, no opcional. El despliegue directo a IIS sigue fuera del alcance obligatorio del MVP.
