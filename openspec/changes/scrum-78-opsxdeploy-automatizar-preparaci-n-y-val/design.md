## Problem Statement

Descripción: Se requiere crear una herramienta interna de despliegue para IIS, similar en estilo operativo a opsxj, que permita preparar, publicar y validar aplicaciones Core a partir de una carpeta de publish generada desde Visual Studio o dotnet publish. Actualmente la publicación a IIS depende de pasos manuales: revisión de archivos publicados, limpieza de artefactos no productivos, creación de carpetas, configuración de Application Pool, creación del sitio, permisos NTFS, configuración de variables de entorno y validación de arranque. Ese proceso es repetitivo, propenso a errores y riesgoso en temas de seguridad, especialmente por exposición accidental de secretos o publicación de archivos de desarrollo. La nueva herramienta debe estandarizar este flujo y bloquear despliegues inseguros. Objetivo: Automatizar el flujo de publicación en IIS de una aplicación Core, incluyendo validaciones de seguridad, preparación de infraestructura local del servidor, despliegue de archivos y verificación operativa básica. Resultado esperado: Disponer de un comando tipo opsxdeploy que permita: validar la carpeta publicada crear estructura de carpetas crear y configurar App Pool / sitio IIS asignar permisos copiar publicación configurar web.config validar arranque del sitio Justificación: reduce errores manuales mejora seguridad de despliegue estandariza publicación entre ambientes acelera despliegues repetitivos evita publicar secretos y archivos no productivos deja trazabilidad de un proceso operativo hoy disperso Alcance MVP: comando doctor comando prepare comando publish Alcance fase 2: comando verify comando full soporte --whatIf validación opcional de endpoint healthcheck generación de reporte final PASS/WARN/FAIL Comandos objetivo: opsxdeploy:doctor opsxdeploy:prepare opsxdeploy:publish opsxdeploy:verify opsxdeploy:full Parámetros esperados: site appPool publishPath sitePath dataPath bindingProtocol bindingPort bindingHost environment healthUrl whatIf force El sistema debe validar que la carpeta de publicación exista y contenga *.dll, *.deps.json, *.runtimeconfig.json y web.config. El sistema debe detectar y bloquear por defecto publicaciones que incluyan appsettings.Development.json. El sistema debe detectar y bloquear por defecto publicaciones que incluyan archivos o carpetas no productivas configuradas como exclusiones. El sistema debe advertir o bloquear si detecta secretos evidentes en appsettings.json. El sistema debe crear la carpeta física del sitio si no existe. El sistema debe crear la estructura de carpetas de datos operativos si no existe. El sistema debe crear o reutilizar un Application Pool configurado en No Managed Code. El sistema debe crear o reutilizar un sitio IIS con ruta física y binding configurables. El sistema debe asignar permisos NTFS al IIS AppPool\<NombrePool> sobre la carpeta de datos operativos. El sistema debe copiar la publicación al directorio destino del sitio. El sistema debe actualizar o generar web.config con variables de entorno para producción. El sistema debe soportar configuración de ConnectionStrings, Jwt, StoragePaths y otras variables necesarias mediante environmentVariables. El sistema debe emitir un resumen final con resultado PASS, WARN o FAIL. En una fase posterior, el sistema debe poder verificar que el sitio arranca y responde en un endpoint configurable. El sistema debe soportar un modo --whatIf o equivalente para simular la ejecución sin aplicar cambios. Subtareas sugeridas: Diseñar contrato del comando opsxdeploy Crear estructura base Tools/iis-deploy Implementar parser de argumentos y logging común Implementar validaciones de carpeta de publicación Implementar validación de secretos y archivos prohibidos Implementar creación de carpetas de sitio y data Implementar creación/configuración de Application Pool Implementar creación/configuración de sitio IIS Implementar asignación de permisos NTFS Implementar copia de publicación al destino Implementar generación/actualización de web.config Implementar comando doctor Implementar comando prepare Implementar comando publish Implementar comando verify Implementar comando full Documentar uso en README Agregar ejemplos operativos para DocuArchi.Api Riesgos: crear o modificar sitios IIS incorrectos por parámetros erróneos sobreescribir un despliegue existente manejar secretos de manera insegura diferencias entre servidores por versión de runtime o permisos publicar una carpeta que aparentemente arranca pero no está lista funcionalmente Definición de terminado: herramienta ejecutable desde npm/PowerShell README con ejemplos reales validaciones de seguridad funcionando despliegue de prueba exitoso sobre una app Core del entorno salida de consola clara y accionable
